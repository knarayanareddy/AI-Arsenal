# PR: Trending vertical reorganisation (`entry_type: trend`)

> Implements the **Trending Vertical Reorganisation & Authoring Agent Prompt**.
> This promotes Trending from `guide` + `section: trending` entries into a
> first-class `entry_type: "trend"` vertical with its own schema, taxonomy
> vocabulary, generation/migration scripts, and CI wiring.

## What this adds

- **New `entry_type: trend`** validated by `schemas/trend.schema.json`, replacing the previous `guide` + `section: trending` convention (the old `content/trending/_index.md` claim that "no new schema is required" is now corrected).
- **Three `kind`s**: `weekly-snapshot`, `hall-of-fame`, `source-feed`.
- **+4 trend entries** (all live-verified, `as_of: 2026-07-06`):
  - `content/trending/this-week.md` — `weekly-snapshot` (auto-drafted by `draft-trending.js`)
  - `content/trending/hall-of-fame.md` — `hall-of-fame`
  - `content/trending/by-source/github-trending.md` — `source-feed`
  - `content/trending/by-source/toolradar-techpresso.md` — `source-feed`
- **Taxonomy vocabulary** (`TAXONOMY.md`): `Trend Kinds`, `Trend Status`, `Trend Signal Types`, `Trend Sources`.
- **Migration guard** `scripts/check-trending-migration-progress.js` (+ `migration:trending:progress` / `migration:trending:enforce` npm scripts, wired into `ci`).
- **`draft-trending.js`** rewritten to emit the `trend` schema (weekly-snapshot with `window`, `signals_used`, `sources`, `ranked_entries`).
- **Phase 0 gitignore/gitattributes negations** so `content/trending/_index.md` and `content/trending/by-source/_index.md` are trackable (verified via `git check-ignore`).

## Notes / intent

- **No undated trending claims.** Every snapshot carries `as_of` (2026-07-06) and dated `sources[].last_checked`; `weekly-snapshot` additionally carries a `window` (7-day span).
- **No fabricated numbers.** `ranked_entries[].score_snapshot` is omitted from hand-authored entries (the auto-draft `this-week.md` derives it from real `trending_score` in `data/projects.json`). `why_here` is mechanism-based, never vibes.
- **No placeholder URLs.** All cited source URLs were live-verified on **2026-07-06** (see completion report).
- **`data/trending.json` is gitignored** (not in the committed allow-list) per repo policy — it is regenerated by `generate:all`.
- **Phase 0 negations are bundled** with the rest of the change set. Per the original brief these would be commit #1; in this environment commits are not possible, so all changes are left as an uncommitted working-tree set (see "Follow-ups").
