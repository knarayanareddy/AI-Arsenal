## Overview

candle-vllm is a Rust implementation of local language-model serving that uses Hugging Face Candle and presents an OpenAI-compatible API. It targets engineers who want inference in a Rust process or binary, rather than a Python runtime surrounding a native backend.

## Why it's in the Arsenal

It deserves a slot as a technically distinct serving path: Candle gives Rust developers a readable tensor and model implementation foundation, while the API lets existing OpenAI clients connect without a protocol rewrite. The MIT license is also friendly to embedding and adaptation.

## Architecture

The project combines Candle model execution with an HTTP server on the documented port and an installer script for setup. A request enters through the OpenAI-compatible route, is tokenized and generated by a supported Candle model, and is returned in the familiar chat/completions shape; the repository tracks supported architectures separately from vLLM.

## Ecosystem Position

candle-vllm complements the Arsenal's Python and MLX inference engines and competes with llama.cpp, mistral.rs, and vLLM for local serving. Its distinguishing boundary is Rust-native integration, while its smaller model matrix means it should not be presented as a general vLLM replacement.

## Getting Started

Run the repository installer at `https://ericlbuehler.github.io/candle-vllm/install.sh` or build the Rust project as documented, select a supported model, and start the server on its default localhost endpoint. Send a chat request with an OpenAI client or curl after checking the model-specific configuration.

## Key Use Cases

Use it inside Rust applications, for local inference services where a Python dependency is undesirable, and for comparing Candle kernels and model implementations. It is also a useful study target for engineers evaluating how model serving maps onto Rust ownership and deployment.

## Strengths

Rust packaging, Candle integration, an OpenAI-compatible API, and MIT licensing make the project easy to embed conceptually and operationally. It avoids requiring the full Python serving ecosystem for a local prototype.

## Limitations

The project has 699 stars and a narrower architecture and quantization surface than mature servers. Throughput, batching, streaming behavior, and production reliability need validation for each model rather than assumption from the vLLM name.

## Relation to the Arsenal

It sits beside vLLM-MLX, Ollama, and other inference engines in the Arsenal, offering a Rust alternative rather than another agent framework. Downstream agent clients can use the same API while operators trade breadth for native Rust control.
