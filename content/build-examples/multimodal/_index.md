# Multimodal Build Examples

## What belongs here

End-to-end blueprints for systems that combine more than one modality: vision+language pipelines, audio+language systems, document-image understanding, image-generation pipelines, and speech systems (ASR/TTS-integrated assistants). The defining trait is that a non-text modality is a first-class input or output of the system being built, not just an incidental file format.

## What does NOT belong here

Parsing PDFs/Office documents into text for a text-only RAG pipeline is a data-pipeline or RAG-system concern (the "multimodal" input is fully converted to text before the interesting engineering happens) — see `data-pipelines/` or `rag-systems/` instead. A single tip about a specific multimodal model or library doesn't belong here unless it is part of a complete, reproducible build.
